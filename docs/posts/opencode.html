在 Windows 下使用 npm 部署 OpenCode

Lagrange2026/4/9大约 8 分钟工具AI开发OpenCodenpmAI辅助

卡戎最近尝试了opencode这个AI编程助手，发现它能在命令行里帮忙写代码，分享一下部署和使用方法喵~

在 Windows 下使用 npm 部署 OpenCode

简介

OpenCode 是一个强大的命令行 AI 编程助手，它可以直接在终端中帮助你完成各种编程任务，比如写代码、调试、重构等。卡戎觉得对于需要快速原型开发或者学习新技术的开发者来说特别有用喵~

注

OpenCode 官方网站：https://opencode.ai/
OpenCode 官方文档：https://opencode.ai/docs/zh-cn/

前期准备

提示

本文基于 Windows 系统进行部署，在其它系统可能会有不一样的地方喵~

1. 安装 Node.js 和 npm

这是使用 OpenCode 的基础环境，就像搭建天文台需要先有地基一样喵~

  1. 访问 Node.js 官网
    打开浏览器，访问 Node.js 官方网站。

  2. 下载安装程序

    • 网站会自动检测你的系统，推荐下载 LTS（长期支持）版本
    • 点击下载按钮，获得 Windows 安装包（.msi 文件）

  3. 安装 Node.js

    • 运行下载的安装包
    • 按照安装向导的提示完成安装
    • 建议保持默认设置，安装 npm 包管理器

  4. 验证安装
    安装完成后，打开 PowerShell 或命令提示符，输入以下命令验证安装：

PowerShell
node -v
npm -v
PowerShell Reply
v20.15.0  # Node.js 版本号，你的可能不同
10.7.0    # npm 版本号，你的可能不同

2. 配置 npm（可选）

如果你在中国大陆，可以配置 npm 镜像源来加速下载：

npm config set registry https://registry.npmmirror.com

安装 OpenCode

1. 全局安装 OpenCode

打开 PowerShell 或命令提示符，运行以下命令：

npm i -g opencode-ai

2. 验证安装

安装完成后，验证 OpenCode 是否安装成功：

opencode --version

如果看到版本号输出，说明安装成功喵~

相关信息

如果遇到权限问题，可以尝试使用管理员权限运行 PowerShell，或者使用以下命令：

npm i -g opencode-ai --unsafe-perm=true --allow-root

3. 其他安装方式

根据 OpenCode 官方文档，还有多种安装方式可选：

安装脚本（推荐）
curl -fsSL https://opencode.ai/install | bash
Homebrew (macOS/Linux)
brew install anomalyco/tap/opencode
Chocolatey (Windows)
choco install opencode
Scoop (Windows)
scoop install opencode
Bun
bun install -g opencode-ai
pnpm
pnpm install -g opencode-ai
Yarn
yarn global add opencode-ai
Docker
docker run -it --rm ghcr.io/anomalyco/opencode

4. Windows 系统建议

对于 Windows 用户，官方推荐使用 Windows Subsystem for Linux (WSL) 以获得最佳体验，因为 WSL 提供更好的性能并完全兼容 OpenCode 的所有功能喵~

OpenCode 基础使用

1. 启动 OpenCode

在项目目录中打开终端，直接运行：

opencode

第一次运行时会提示你进行身份验证，按照提示完成即可。

2. 初始化项目

首次在项目中运行 OpenCode 时，建议使用 /init 命令初始化项目：

/init

这个命令会分析你的项目结构，并在项目根目录创建一个 AGENTS.md 文件，帮助 OpenCode 理解项目规范。建议将此文件提交到 Git 仓库中喵~

3. 基本命令

注

以下是一些常用的 opencode 命令示例：

帮助命令 - 查看所有可用命令：

opencode --help

提问模式 - 直接向 OpenCode 提问：

opencode "如何实现一个 REST API？"

代码审查 - 让 OpenCode 检查当前目录的代码：

opencode review

文件操作 - 让 OpenCode 处理特定文件：

opencode file.js

4. 交互模式

OpenCode 支持交互式对话，就像和一个编程助手聊天一样：

opencode chat

进入交互模式后，你可以：

  • 输入 /help 查看帮助
  • 输入 /connect 快速配置 API 连接
  • 输入 /exit 退出聊天
  • 输入 /init 初始化项目（首次使用时）
  • 输入 /undo 撤销上一次修改
  • 输入 /redo 重做被撤销的修改
  • 输入 /share 生成对话分享链接
  • 使用 @ 键模糊搜索项目中的文件（如 @src/components/Button.tsx）
  • 按 Tab 键在计划模式（只生成计划）和构建模式（直接修改代码）之间切换
  • 直接将图片拖拽到终端中，OpenCode 可以识别图片内容作为参考
  • 直接输入问题或代码片段进行讨论

使用示例

示例 1：创建简单的 React 组件

opencode "创建一个简单的 React 计数器组件"

示例 2：调试现有代码

假设你有一个 buggy.js 文件有问题：

opencode buggy.js

OpenCode 会分析代码并提出修复建议。

示例 3：重构代码

opencode "重构这个函数，提高可读性" < your-function.js

连接 DeepSeek API

OpenCode 支持连接外部 AI API，比如 DeepSeek 开放平台。这样你可以使用自己的 API 密钥，获得更稳定和个性化的 AI 协助喵~

1. 获取 DeepSeek API 密钥

  1. 访问 DeepSeek 开放平台
    打开浏览器，访问 DeepSeek 开放平台。

  2. 注册/登录账户

    • 如果你还没有账户，点击注册并完成验证
    • 如果已有账户，直接登录

  3. 创建 API 密钥

    • 登录后，进入 "API Keys" 页面
    • 点击 "Create new secret key"
    • 为密钥设置一个描述性的名称（例如 "OpenCode-Desktop"）
    • 复制生成的 API 密钥并妥善保存

注意

API 密钥就像天文台的钥匙一样重要，不要分享给他人，也不要提交到代码仓库中喵~

2. 配置 OpenCode 使用 DeepSeek API

OpenCode 支持通过环境变量配置 AI 提供商。以下是配置方法：

方法一：通过环境变量设置

在 PowerShell 中设置环境变量：

$env:OPENCODE_API_KEY = "你的-DeepSeek-API-密钥"
$env:OPENCODE_API_BASE = "https://api.deepseek.com"
$env:OPENCODE_MODEL = "deepseek-chat"

方法二：创建配置文件

在你的用户目录下创建 ~/.opencode/config.json 文件：

{
  "apiKey": "你的-DeepSeek-API-密钥",
  "apiBase": "https://api.deepseek.com",
  "model": "deepseek-chat"
}

方法三：在会话中临时设置

启动 OpenCode 时直接指定参数：

opencode --api-key "你的-DeepSeek-API-密钥" --api-base "https://api.deepseek.com" --model "deepseek-chat"

方法四：使用 /connect 命令快速连接

OpenCode 提供了更便捷的 /connect 命令，可以在交互模式中快速设置 API 连接：

  1. 启动交互模式

    opencode chat

  2. 使用 /connect 命令
    在交互模式中输入：

    /connect

    OpenCode 会引导你完成连接设置：

    • 输入 API 密钥
    • 选择或输入 API 提供商（DeepSeek）
    • 设置模型名称（如 deepseek-chat）
    • 确认配置

  3. 保存配置（可选）
    连接成功后，可以输入：

    /save-config

    将当前配置保存到 ~/.opencode/config.json，下次启动时自动使用。

3. 验证连接

配置完成后，启动 OpenCode 并询问一个简单问题来验证连接：

opencode "你好，测试 API 连接"

如果看到正常的 AI 回复，说明连接成功喵~

4. 高级配置选项

DeepSeek API 支持一些额外的参数配置：

# 设置温度参数（控制回复的随机性，0-2之间）
$env:OPENCODE_TEMPERATURE = "0.7"

# 设置最大 tokens（控制回复长度）
$env:OPENCODE_MAX_TOKENS = "2000"

# 启用流式响应
$env:OPENCODE_STREAM = "true"

5. 使用示例

示例 1：使用自定义 API 的代码生成

opencode --api-key "你的密钥" "用 Python 实现快速排序算法"

示例 2：带温度控制的代码审查

opencode --temperature 0.3 review src/

示例 3：长回复的流式输出

opencode --stream "详细解释 JavaScript 的闭包概念"

常见问题

Q1：安装时遇到权限错误

解决方法：

  • 使用管理员权限运行终端
  • 或者修改 npm 全局安装目录的权限
  • 或者使用 npm i -g opencode-ai --unsafe-perm=true

Q2：OpenCode 响应慢

可能原因：

  • 网络连接问题
  • AI 模型正在处理复杂请求

解决方法：

  • 检查网络连接
  • 尝试简化问题描述
  • 使用更具体的命令

Q3：如何更新 OpenCode

npm update -g opencode-ai

建议

提示

  1. 明确问题 - 给 OpenCode 的问题越具体，得到的回答越准确，就像向望远镜描述要观察的星体一样喵~
  2. 提供上下文 - 为 OpenCode 提供充足的上下文和示例，帮助它更好地理解你的需求
  3. 使用计划模式 - 对于复杂任务，先用 Tab 键切换到计划模式，审查计划后再切换到构建模式执行
  4. 逐步交互 - 复杂任务可以分解成多个小问题，逐步与 OpenCode 交互
  5. 结合文档 - OpenCode 的回答可以作为参考，但重要功能还是要查看官方文档
  6. 安全注意 - 不要通过 OpenCode 处理敏感信息或密钥

总结

OpenCode 对于卡戎这样的天文台管理员来说是个不错的编程助手，通过简单的命令行交互就能获得 AI 的编程帮助。对于日常开发、学习新技术或者快速解决问题都有很好的辅助作用。虽然不能完全替代人类的思考，但在很多场景下能大大提高效率喵~

希望这个教程能帮助你顺利部署和使用 OpenCode！如果在使用过程中遇到问题，可以查看 OpenCode 官方文档 或社区讨论。祝编码愉快~ 🚀
