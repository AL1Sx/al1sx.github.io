WinUI3 初次接触

WinUI3 是什么

WinUI 3 是微软推出的现代 Windows 桌面应用 UI 框架，属于 Windows App SDK 的一部分。简单来说，它是 WPF 和 UWP 的"继任者"，用 C# 或 C++ 配合 XAML 来构建界面，支持 Fluent Design（微软的现代设计语言）。

注

WinUI3 官方文档
Windows App SDK 文档
WinUI 3 Gallery（官方示例）
WinUI 说明教程

相比之前的框架，WinUI3 的几个主要优势：

  • 原生支持 Fluent Design，控件风格和 Windows 11 一致
  • 支持 Mica 亚克力等窗口背景效果
  • 基于 .NET 8，性能和现代 C# 特性都有保障
  • 最低支持 Windows 10 1809（build 17763）

开发环境

提示

本文基于 Windows 11 + Visual Studio 2026 进行开发。

需要安装的东西：

软件说明
Visual Studio 2026需要勾选「WinUI 应用程序开发」工作负载
.NET 8 SDK项目目标框架
Windows 10/111809 及以上版本
开发人员模式Windows 设置中开启

快速安装（推荐）

微软提供了一个 WinGet 配置文件，可以一键安装所有需要的工具：

winget configure -f https://aka.ms/winui-config

手动安装

  1. 开启开发人员模式：Windows 设置 → 系统 → 高级 → 开发人员模式 → 开
  2. 下载安装 Visual Studio 2026
  3. 在 VS 安装程序中勾选 WinUI 应用程序开发 工作负载

注

快速入门：创建第一个 WinUI 3 应用

创建项目

在 Visual Studio 中创建 WinUI3 项目：新建项目 → 搜索 "WinUI" → 选择 WinUI 空白应用（打包）。

也可以用命令行（需要先安装模板）：

# 安装 WinUI 项目模板（只需执行一次）
dotnet new install Microsoft.WindowsAppSDK.WinUI.CSharp.Templates

# 创建项目
dotnet new winui -n MyApp
cd MyApp

# 运行
dotnet run

注

VS 的 XAML Hot Reload 功能很好用，运行时修改 XAML 可以实时看到效果，不用重启应用。
用于 Windows 应用开发的 Visual Studio

项目结构

一个基本的 WinUI3 项目结构如下：

MyApp/
├── App.xaml(.cs)           # 应用入口
├── MainWindow.xaml(.cs)    # 主窗口
├── Assets/                 # 图标、图片资源
├── Package.appxmanifest    # 应用清单（权限、图标等）
├── Properties/             # 发布配置
└── MyApp.csproj            # 项目文件

App.xaml.cs 是应用的启动入口，MainWindow.xaml 是主窗口的布局文件。如果要做多页面导航，通常会在这里放一个 NavigationView。

关键概念

XAML 数据绑定

WinUI3 使用 XAML 做界面标记，推荐用 x:Bind（编译时绑定）而不是传统的 Binding，性能更好：

<TextBlock Text="{x:Bind ViewModel.CpuUsage, Mode=OneWay}" />

MVVM 模式

推荐使用 MVVM 模式组织代码。CommunityToolkit.Mvvm 提供了很方便的命令支持：

using CommunityToolkit.Mvvm.ComponentModel;
using CommunityToolkit.Mvvm.Input;

public partial class MyViewModel : ObservableObject
{
    [RelayCommand]
    private void DoSomething() { ... }
}

警告

CommunityToolkit.Mvvm 的 [ObservableProperty] 在 WinUI3 的 XAML 编译器中可能无法正常工作（源生成器生成的属性在编译时不可见）。如果遇到绑定失败，需要手动实现 INotifyPropertyChanged。

NavigationView 导航

多页面应用通常用 NavigationView + Frame 实现：

<NavigationView x:Name="NavView"
                SelectionChanged="NavView_SelectionChanged">
    <Frame x:Name="ContentFrame" />
    <NavigationView.MenuItems>
        <NavigationViewItem Content="首页" Tag="Home">
            <NavigationViewItem.Icon>
                <FontIcon Glyph="&#xE80F;" />
            </NavigationViewItem.Icon>
        </NavigationViewItem>
    </NavigationView.MenuItems>
</NavigationView>

Mica 背景

WinUI3 支持 Mica（云母）材质背景，让窗口和桌面融为一体：

<Window.SystemBackdrop>
    <MicaBackdrop />
</Window.SystemBackdrop>

踩坑记录

后台线程更新 UI 不生效

WinUI3 的属性绑定需要在 UI 线程上触发才能生效。如果在后台线程调用 OnPropertyChanged，UI 不会更新。

解决方法：用 DispatcherQueue 调度到 UI 线程：

private readonly DispatcherQueue? _dispatcher;

public MyViewModel()
{
    _dispatcher = DispatcherQueue.GetForCurrentThread();
}

private void UpdateUI(Action action)
{
    _dispatcher?.TryEnqueue(() => action());
}

XAML 转换器无法解析

XAML 编译器有时无法解析本地命名空间中的转换器类型。解决方法是在 App.xaml.cs 的 OnLaunched 中动态注册：

protected override void OnLaunched(LaunchActivatedEventArgs args)
{
    Resources["MyConverter"] = new MyConverter();
    // ...
}

Button 不支持 Icon 属性

WinUI3 的 Button 和 WPF 不同，没有 Icon 属性。需要用 Content 组合图标和文字：

<Button>
    <StackPanel Orientation="Horizontal" Spacing="8">
        <FontIcon Glyph="&#xE9F5;" FontSize="14" />
        <TextBlock Text="开始" />
    </StackPanel>
</Button>

PerformanceCounter 找不到

.NET 8 中 System.Diagnostics.PerformanceCounter 被转发到了独立程序集。需要手动安装 NuGet 包：

dotnet add package System.Diagnostics.PerformanceCounter

独立发布的坑

WinUI3 支持发布为独立可执行文件（不需要用户安装 .NET 运行时），但有几个要注意的地方：

  • 必须在 .csproj 中设置 WindowsAppSDKSelfContained=true
  • 不要使用 PublishSingleFile=true，会导致 WinUI3 应用崩溃（0xc000027b）
  • 如果用了 System.Management 或 PerformanceCounter，不要开启裁剪（PublishTrimmed=false），否则这些功能会失效
  • 独立发布的文件会比较大（约 300MB），这是因为包含了完整的 .NET 运行时和 Windows App SDK，属于正常现象

注

发布命令：

dotnet publish -p:Platform=x64 -p:Configuration=Release

构建和运行

# 调试运行
dotnet run -p:Platform=x64

# 构建
dotnet build -p:Platform=x64

# 发布独立可执行文件
dotnet publish -p:Platform=x64 -p:Configuration=Release

注意

如果项目中使用了 PerformanceCounter，必须指定 -p:Platform=x64，否则会报错。

总结

WinUI3 作为微软最新的 Windows 桌面 UI 框架，整体开发体验还是不错的。Fluent Design 的控件很好看，Mica 背景也让窗口看起来更现代化。不过从 WPF 或 UWP 转过来的话，还是有不少地方需要注意，比如后台线程更新 UI、XAML 绑定方式的差异等。

最大的坑应该是独立发布——文件体积大、不能用 PublishSingleFile、裁剪会导致功能失效。如果只是个人项目或者内部使用，这些问题还好；如果要公开分发，还需要考虑代码签名的问题。

总的来说，WinUI3 值得一试，特别是对于想做现代 Windows 桌面应用的开发者。
