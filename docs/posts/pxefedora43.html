现代设备 PXE 网络安装 Fedora 43 完整教程（Windows 做服务器）

卡戎最近尝试了PXE网络安装Fedora 43，发现有两种不错的方案，分享一下喵~

现代设备 PXE 网络安装 Fedora 43 完整教程（Windows 做服务器）

适用环境：Windows 11 作为 PXE 服务器，目标设备支持 UEFI、千兆/2.5G 网络。
最终效果：通过网络引导 Fedora 43 Live 环境，并安装到硬盘。
提供 Tiny PXE Server（灵活可控）和 iVentoy（开箱即用）两种方案，任选其一即可喵~

通用准备工作（两种方案都需要）

提示

在进行PXE安装前，请确保完成以下准备工作，就像搭建天文台前要先检查仪器一样重要喵~

  1. 一台 Windows 电脑（服务器），设置静态 IP（例如 192.168.3.10）。
  2. 待安装设备（现代 PC/笔记本），与服务器接入同一交换机/路由器。
  3. Fedora 43 Workstation Live ISO：从 官网 下载 Fedora-Workstation-Live-43-1.6.x86_64.iso。
  4. 关闭防火墙（或后续放行相应端口）：建议先临时关闭 Windows 防火墙进行测试，成功后再配置入站规则。

方案一：Tiny PXE Server（灵活可控，适合进阶用户）

1. 下载与安装

  • 从 GitHub 下载 Source code zip 包（Releases 中通常没有 exe，需要下载源码包）。
  • 解压到不含中文和空格的路径，例如 C:\PXE。
  • 解压后自带 files 目录（例如 C:\PXE\files\），无需手动创建。

2. 准备所需文件

将以下文件 全部放入 C:\PXE\files\ 目录中：

文件来源
Fedora-Workstation-Live-43-1.6.x86_64.iso从 Fedora 官网下载
ipxe.efi从 https://ipxe.org/download 下载（UEFI 启动用）
vmlinuz 和 initrd.img从 ISO 中提取（见下方说明）

如何提取 vmlinuz 和 initrd.img？

  • 用 7-Zip 打开 Fedora 43 ISO。
  • 进入 /images/pxeboot/ 目录。
  • 将 vmlinuz 和 initrd.img 复制到 C:\PXE\files\。

3. 编写 iPXE 启动菜单

在 C:\PXE\files\ 目录下新建文本文件 menu2.ipxe（名字可自定义），用记事本打开，写入以下内容（将 192.168.3.10 替换为你的服务器真实 IP）：

#!ipxe

menu Fedora 43 NetBoot Menu

item fedora-live Boot Fedora 43 Workstation Live
item shell Drop to iPXE shell
item reboot Reboot

choose --timeout 10000 --default fedora-live selected
goto ${selected}

:fedora-live
kernel http://192.168.3.10/vmlinuz initrd=initrd.img
initrd http://192.168.3.10/initrd.img
boot

:shell
shell
goto menu

:reboot
reboot

注

initrd=initrd.img 是内核参数，告诉内核使用哪个初始内存盘，就像告诉望远镜用哪个滤镜一样喵~

4. 启动 Tiny PXE Server

  1. 以管理员身份运行 C:\PXE\pxesrv.exe。
  2. 配置界面：
    • Server Interface：选择你的本机 IP（如 192.168.3.10）。
    • Boot File：填入 ipxe.efi。
    • 勾选 Proxy DHCP（重要，避免与路由器 DHCP 冲突）。
    • 勾选 HTTPd（加速文件传输）。
    • Boot Menu：填入 menu2.ipxe。
  3. 点击 Online 按钮。日志窗口显示 DHCPd、TFTPd、HTTPd 即成功。

5. 客户端引导与安装

  1. 在待安装设备上开机，进入 UEFI 设置（通常按 F2 / Del / F12）：
    • 关闭 Secure Boot（安全启动）。
    • 启动模式设为 UEFI（不要 Legacy）。
    • 将 网络启动（PXE / Network Boot） 设为第一启动项。
  2. 保存重启，设备会自动获取 IP 并显示 iPXE 菜单。
  3. 选择 Boot Fedora 43 Workstation Live，系统将启动进入 Fedora Live 桌面。
  4. 在 Live 桌面中安装系统：
    • 双击桌面上的 "Install to Hard Drive" 图标，或
    • 右键打开终端，输入 liveinst（不需要 sudo）并回车。
  5. 按照安装向导完成分区、用户设置等步骤。
  6. 安装完成后重启，拔掉 PXE 启动（或修改 BIOS 启动顺序为硬盘）。

方案二：iVentoy（开箱即用，适合新手）

1. 下载与安装

  • 从 iVentoy 官网 下载 Windows 免费版（iventoy-xxx-win64-free.zip）。
  • 解压到不含中文和空格的路径，例如 D:\iVentoy。

2. 放置 ISO 镜像

将 Fedora-Workstation-Live-43-1.6.x86_64.iso 直接复制到解压后的 iso 文件夹中。

3. 启动 iVentoy

  1. 以管理员身份运行 iVentoy_64.exe。
  2. 程序会自动打开浏览器进入 Web 管理界面（http://127.0.0.1:26000）。
  3. 配置：
    • 选择本机网卡 IP（如 192.168.3.10）。
    • DHCP Server Mode：选择 Proxy（推荐，与路由器和谐共存）。
    • 点击 Start 按钮。
  4. 状态显示"服务运行中"即成功。

4. 客户端引导与安装

  1. 待安装设备：UEFI 关闭 Secure Boot，网络启动设为第一启动项。
  2. 重启后自动进入 iVentoy 引导菜单，选择 Fedora-Workstation-Live-43-1.6.x86_64.iso。
  3. 系统启动进入 Fedora Live 桌面。
  4. 安装系统：双击桌面 "Install to Hard Drive" 图标，或终端运行 liveinst。
  5. 按向导完成安装，重启后从硬盘引导。

常见问题排查（两种方案通用）

警告

如果在安装过程中遇到问题，先检查这些常见情况，就像观察星空时遇到云层一样需要耐心喵~

现象解决方法
客户端卡在 DHCP...关闭 Windows 防火墙；确保 PXE 工具勾选了 Proxy DHCP。
提示 No boot filename received检查启动文件（ipxe.efi）是否在正确目录，文件名是否写对。
加载内核后黑屏在 kernel 参数末尾添加 nomodeset（iVentoy 可在启动项按 e 编辑）。
liveinst 命令报错尝试不加 sudo 运行，或从图形界面启动安装器。
安装时找不到硬盘进 UEFI 将 SATA 模式改为 AHCI（而非 RAID / Intel RST）。
Secure Boot 导致无法引导进 UEFI 关闭 Secure Boot。

进阶：使用网络安装镜像（Netinstall）

提示

Live 镜像会加载整个桌面环境，内存占用较高。如果希望直接进入安装程序，可以使用 Fedora 的 Everything netinstall ISO（约 700MB），就像选择更精确的观测工具一样喵~

  • 下载 Fedora-Everything-netinst-x86_64-43-1.6.iso。
  • Tiny PXE Server 用户：
    • 提取 /images/pxeboot/ 下的 vmlinuz 和 initrd.img。
    • 修改 menu2.ipxe 内核参数：
      kernel http://192.168.3.10/vmlinuz inst.repo=http://192.168.3.10
initrd http://192.168.3.10/initrd.img
    • 将 ISO 内容解压到 files 目录（确保 repodata 可被 HTTP 访问）。
  • iVentoy 用户：直接放入 iso 文件夹即可，无需额外配置。

注

卡戎觉得，就像选择天文观测工具一样，根据你的需求和技术水平选择合适的方案最重要喵~

总结

无论选择哪种方案，核心流程都是：

  1. 准备服务器：放置 ISO 和相关文件，启动 PXE 服务。
  2. 配置客户端：UEFI 关闭 Secure Boot，网络启动优先。
  3. 引导进入 Live 环境：通过菜单选择 Fedora 43。
  4. 执行安装：运行 liveinst 或点击桌面安装图标。
  5. 重启完成：从硬盘引导进入新系统。

本教程基于 Windows 11 + Tiny PXE Server / iVentoy + Fedora 43 实测成功，适用于现代 UEFI 设备。如果在安装过程中遇到其他问题，可以像探索未知星体一样耐心排查，相信你也能成功完成安装喵~
